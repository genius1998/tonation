# React + Web Speech API 개발 문제 해결 요청

저는 현재 React와 Node.js(Socket.io)를 사용하여 인터넷 방송용 후원 알림 오버레이(Overlay) 시스템을 개발 중입니다.
OBS Browser Source에서 사용할 목적으로, 로컬 브라우저의 Web Speech API(TTS)와 HTML5 Audio를 사용하고 있습니다.

현재 두 가지 치명적인 문제가 해결되지 않고 있어 코드 리뷰와 수정 방안을 요청합니다.

---

## [문제 상황 1] 오디오/TTS 재생 시간 제한(Duration) 기능이 작동하지 않음

**요구사항:**
- 관리자 페이지에서 프리셋별로 '표시 시간(duration)'을 초 단위로 설정합니다. (예: 3초)
- 오버레이에서는 설정된 시간이 지나면 재생 중인 MP3 오디오나 TTS를 즉시 강제 종료하고 화면을 닫아야 합니다.
- 만약 시간이 '0'이면 오디오/TTS가 끝날 때까지 재생합니다.

**현재 구현 (Overlay.jsx):**
- `duration > 0`인 경우, `playSequence()`(재생 로직)를 `await` 없이 실행하고, 별도로 `setTimeout`을 걸어 시간 종료 시 `stopAllMedia()`를 호출합니다.
- `stopAllMedia`는 `audio.pause()`와 `speechSynthesis.cancel()`을 수행하고, `canceledRef.current = true`를 설정합니다.

**증상:**
- 시간을 3초로 설정해도, 10초짜리 노래가 끝까지 다 재생된 후에야 알림이 꺼집니다.
- 마치 `setTimeout`이 무시되거나, `playSequence` 내부의 `await`가 강제 종료를 막고 있는 것처럼 보입니다.

## [문제 상황 2] TTS 기본 목소리가 'Google 한국의'로 잡히지 않고 'Microsoft Heami'로 출력됨

**요구사항:**
- 사용자가 특정 목소리를 선택하지 않은 경우(Default), 브라우저에 설치된 목소리 중 **"Google 한국"**이 포함된 목소리를 최우선으로 사용해야 합니다.
- 그 다음으로 "Google" + "ko", 그 다음으로 "Microsoft" + "ko" 순으로 Fallback 해야 합니다.

**현재 구현 (Overlay.jsx):**
- `playTTS` 함수 내에서 `voices.find(v => v.name.includes('Google 한국') && v.lang.includes('ko'))` 로직을 사용 중입니다.
- `onvoiceschanged` 이벤트를 사용하여 목소리 목록 로딩을 기다리는 처리도 되어 있습니다.

**증상:**
- 크롬 브라우저나 OBS에서 테스트 시, 분명히 "Google 한국의..." 목소리가 존재함에도 불구하고(미리듣기에서는 됨), 실제 알림 출력 시에는 "Microsoft Heami"가 출력됩니다.
- `getVoices()` 타이밍 문제인지, 매칭 로직 문제인지 확실치 않습니다.

## [현재 코드: Overlay.jsx]

아래는 현재 문제가 발생하고 있는 `Overlay.jsx`의 핵심 코드입니다.

```jsx
import React, { useState, useEffect, useRef } from 'react';
import io from 'socket.io-client';
import './ToonationStyle.css';

const socket = io('http://localhost:3001');

const Overlay = () => {
  const [queue, setQueue] = useState([]);
  const [currentAlert, setCurrentAlert] = useState(null);
  const [isVisible, setIsVisible] = useState(false);
  
  const audioRef = useRef(new Audio());
  const processingRef = useRef(false);
  const canceledRef = useRef(false);

  // ... (Socket 연결 로직 생략) ...

  const stopAllMedia = () => {
      audioRef.current.pause();
      audioRef.current.currentTime = 0;
      window.speechSynthesis.cancel();
  };

  const processNextAlert = async () => {
    processingRef.current = true;
    canceledRef.current = false;

    const alertData = queue[0];
    setQueue((prev) => prev.slice(1));
    setCurrentAlert(alertData);
    setIsVisible(true);

    // ... (템플릿 변수 파싱 로직 생략) ...

    const soundType = alertData.soundType || (alertData.audioSrc ? 'file' : 'none');
    const ttsConfig = alertData.ttsConfig || {};
    const durationSec = alertData.duration || 0; // Admin에서 넘어온 초 단위 시간 (예: 5)

    const playSequence = async () => {
        if (canceledRef.current) return;

        // 1. 메인 사운드 재생
        if (soundType === 'file' && alertData.audioSrc) {
            audioRef.current.src = alertData.audioSrc;
            try { await audioRef.current.play(); } catch (e) { console.error(e); }
            
            // [의심 구간] 여기서 이벤트 리스너가 강제 종료 시에도 해제되지 않거나 대기하는지?
            await new Promise(resolve => {
                const endHandler = () => {
                    audioRef.current.removeEventListener('ended', endHandler);
                    resolve();
                };
                audioRef.current.addEventListener('ended', endHandler);
            });
        } else if (soundType === 'tts') {
            await playTTS(ttsTemplate, ttsConfig);
        } else {
            await new Promise(r => setTimeout(r, 3000));
        }

        if (canceledRef.current) return;

        // 2. 메시지 TTS 재생
        if (messageContent) {
             await playTTS(messageContent, ttsConfig);
        }
    };

    if (durationSec > 0) {
        // [문제 1 발생 구간] 시간 제한 모드
        // playSequence를 비동기로 시작
        playSequence(); 
        
        // 지정된 시간 대기
        await new Promise(r => setTimeout(r, durationSec * 1000));
        
        // 시간 종료 후 강제 중단
        canceledRef.current = true;
        stopAllMedia();
        
    } else {
        // 자동 길이 모드
        await playSequence();
        await new Promise(r => setTimeout(r, 1000));
    }

    setIsVisible(false);
    await new Promise(r => setTimeout(r, 500));
    setCurrentAlert(null);
    processingRef.current = false;
  };

  const playTTS = (text, config = {}) => {
    return new Promise((resolve) => {
      window.speechSynthesis.cancel();
      const utterance = new SpeechSynthesisUtterance(text);
      // ... (rate, pitch 설정) ...

      const speak = () => {
          if (canceledRef.current) { resolve(); return; }
          const voices = window.speechSynthesis.getVoices();
          let selectedVoice = null;

          // 1. 설정된 이름 찾기
          if (config.voiceName) selectedVoice = voices.find(v => v.name === config.voiceName);
          
          // 2. [문제 2 발생 구간] 디폴트 찾기
          if (!selectedVoice) selectedVoice = voices.find(v => v.name.includes('Google 한국') && v.lang.includes('ko'));
          if (!selectedVoice) selectedVoice = voices.find(v => v.name.includes('Microsoft') && v.lang.includes('ko'));

          if (selectedVoice) utterance.voice = selectedVoice;

          utterance.onend = () => resolve();
          window.speechSynthesis.speak(utterance);
      };

      if (window.speechSynthesis.getVoices().length === 0) {
          window.speechSynthesis.onvoiceschanged = () => { speak(); };
      } else {
          speak();
      }
    });
  };

  // ... (render 로직) ...
};
```

## [요청 사항]

1. **Duration 문제 해결:** `duration > 0`일 때, `playSequence` 내부의 `await audio...`가 끝나지 않았더라도 `stopAllMedia`가 호출되면 즉시 소리가 꺼지고 다음 로직으로 넘어가도록 코드를 어떻게 수정해야 할까요? (현재는 소리가 안 꺼지는 것 같습니다).
2. **TTS Default Voice 문제 해결:** 크롬/OBS 환경에서 'Google 한국' 음성을 확실하게 디폴트로 잡으려면 매칭 로직을 어떻게 개선해야 할까요? (이름이 정확히 일치하지 않거나 부분 일치 문제일까요?)

위 코드를 기반으로 수정된 `processNextAlert`와 `playTTS` 함수 코드를 제안해주세요.
